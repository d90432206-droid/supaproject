# 制宜電測專案管理系統 (Chuyi System) 功能說明書

這份文件詳細介紹了「制宜電測專案管理系統」的架構與功能，旨在作為生成使用者手冊或技術文件之參考基礎。

---

## 1. 系統概述
本系統是一套專為電測產業設計的專案及工時管理平台，整合了專案進度追蹤、甘特圖視覺化、工時回報以及 AI 數據分析功能。

- **前端技術**：React 19, Vite, TypeScript, TailwindCSS, Recharts
- **後端技術**：Supabase (PostgreSQL, Edge Functions)
- **AI 整合**：Google Gemini (多模型備援機制)
- **通訊整合**：LINE Messaging API (用於查詢與週報推送)

---

## 2. 核心功能模組

### 2.1 儀表板 (Dashboard)
- **系統公告**：管理員可發佈重要訊息，全體成員可見。
- **專案概覽**：顯示目前進行中、已結案專案的統計。
- **工時趨勢**：透過 Recharts 圖表顯示近期工時投入狀況。
- **活動摘要**：顯示最近的工時紀錄。

### 2.2 專案管理 (Project List)
- **專案建立與編輯**：支援設定專案編號、名稱、客戶、預算工時及各部門 (ATS, CHS, CPD, MFG) 的預算配額。
- **狀態切換**：可將專案從「進行中 (Active)」切換至「結案 (Closed)」。
- **刪除保護**：包含二次確認與管理密碼驗證。

### 2.3 WBS 編輯器與甘特圖 (WBS Editor)
- **任務分解**：將專案拆解為細項任務，設定負責人、開始日期、結束日期、預算小時。
- **自動計算**：系統會根據日期與例假日 (Holidays) 自動計算工作天數 (Duration)。
- **進度追蹤**：設定任務進度百分比 (%) 與填寫延遲原因。
- **甘體圖視覺化**：右側即時生成甘特圖，以不同顏色代表不同負責工程師。

### 2.4 工時紀錄 (Time Log)
- **每日回報**：工程師選擇專案與任務，輸入工時與備註。
- **歷史檢索**：可按日期、工程師或專案篩選歷史工時。
- **刪除機制**：支援刪除誤填的資料。

### 2.5 專案總覽 (Project Summary)
- **人力分配表**：表格化呈現各專案目前的總投入工時 vs. 預算工時。
- **超支警示**：若實際工時接近或超過預算，系統會給予提示。

### 2.6 成員管理 (Admin Member Management)
- **權限控制**：區分為「管理員」與「工程師」兩類角色。
- **個人化設定**：
  - **代表色**：用於甘特圖顯示。
  - **部門資訊**：分類統計使用。
  - **LINE ID**：綁定 LINE 帳號，以便接收 AI 推送。

## 3. AI 助理與 LINE 整合

系統後端運行於 Supabase Edge Functions (`timelog-ai`)，具備以下智慧功能：

### 3.1 AI 數據分析
- **自然語言查詢**：透過 LINE 傳送指令（如：「這週 ATS 專案進度」、「王小明的工時統計」），AI 會自動解析意圖並從資料庫抓取數據生成分析報告。
- **延遲預警**：AI 可分析工時日誌，標註超預算風險或顯著的停工異常。

### 3.2 自動週報推送
- **定時觸發**：系統可設定 Cron Job 自動生成全體專案摘要，並推播至管理層的 LINE。

### 3.3 模型備援機制 (Reliability)
- 整合了 Gemini 3.1 Flash Lite, 3 Flash, 2.5 Flash 等多種模型。
- 當遇到 API 繁忙 (503/429) 時，具備自動重試與模型自動降級 (Fallback) 機制。

---

## 4. 資料庫結構 (關鍵資料表)
- `prj_projects`：專案基本資料與 WBS JSON。
- `prj_logs`：工時回報日誌。
- `prj_settings`：系統設定與成員名單（包含 LINE ID 綁定）。
- `prj_messages`：系統公告。

---

## 5. 權限與登入
- 系統採簡單密碼制登入。
- **管理員**：擁有所有專案的建立、編輯、刪除權限，以及成員與公告管理權限。
- **工程師**：以個人帳號登入後，主要執行任務進度更新與工時回報。
